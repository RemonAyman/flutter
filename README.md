# مشروع صلحني (Sal7ny) 🛠️

منصة متكاملة لربط مقدمي الخدمات (الصنايعية) بالعملاء، مبنية بأحدث التقنيات لضمان تجربة مستخدم سلسة واحترافية.

---

## 🏗️ هيكل المشروع (Project Structure)

المشروع مقسم إلى جزئين رئيسيين (Full-Stack):

### 1. الباك إند (Backend - Node.js & Express)
يوجد في مجلد `backend/` ويحتوي على:
- **`index.js`**: نقطة الانطلاق الرئيسية للسيرفر وربط المسارات.
- **`src/models/`**: يحتوي على "موديلات" قاعدة البيانات (User, Worker, Request).
- **`src/routes/`**: يحتوي على منطق التحكم في البيانات (Auth, Admin, Services).
- **`src/config/`**: إعدادات الاتصال بقاعدة البيانات.

### 2. الفرونت إند (Frontend - Flutter)
يوجد في مجلد `te/` ويحتوي على:
- **`lib/pages/`**: جميع شاشات التطبيق مقسمة حسب الأدوار (Admin, Client, Worker).
- **`lib/services/`**: ملف `api_service.dart` المسؤول عن التواصل مع الباك إند.
- **`lib/widgets/`**: المكونات القابلة لإعادة الاستخدام.

## 🧭 رحلة المستخدم (User Journey) بالتفصيل

### أولاً: العميل (Customer/Client) 👤
1. **اختيار الدور**: عند فتح التطبيق، يختار "عميل".
2. **التسجيل/الدخول**: ينشئ حساباً بالاسم والإيميل والباسورد.
3. **الصفحة الرئيسية**: تظهر له قائمة الحرف (كهرباء، نجار، سباك، دهان، مكيفات).
4. **قائمة الصنايعية**: بعد اختيار الحرفة، تظهر له قائمة بالصنايعيين الحقيقيين من قاعدة البيانات مع تقييمهم وأسعارهم.
5. **حجز خدمة**: يضغط على "احجز"، يكتب وصفاً لمشكلته ويختار الوقت المناسب.
6. **التأكيد**: يتم إرسال الطلب فوراً ليظهر في لوحة تحكم الصنايعي المختار.

### ثانياً: مقدم الخدمة (الصنايعي/Worker) 👷
1. **اختيار الدور**: يختار "صنايعي".
2. **التسجيل**: عند تسجيله كصنايعي، يقوم السيستم تلقائياً بإنشاء ملف تعريف له في قاعدة البيانات كـ "مقدم خدمة".
3. **لوحة التحكم (Dashboard)**: تظهر له خيارات واضحة:
   - **مشاهدة الطلبات الجديدة**: لمتابعة طلبات العملاء الحقيقية القادمة من MongoDB.
   - **تحديد مواعيد العمل**: لتنظيم ساعاته المتاحة.
   - **الملف الشخصي**: لعرض بياناته الشخصية وتقييمه.
4. **استلام الطلبات**: يرى اسم العميل، نوع المشكلة، والوقت المختار، ويمكنه قبول الطلب.

### ثالثاً: المدير (Admin) ⚡
- **الدخول**: يتم من خلال حساب موحد من أي صفحة لوجين ببيانات:
  - **الإيميل**: `admin@gmail.com`
  - **الباسورد**: `admin123`
- **لوحة التحكم**: تعرض إحصائيات حقيقية من MongoDB (إجمالي العملاء، إجمالي الصنايعية، وأحدث 5 طلبات تمت في السيستم).

## 💾 الربط مع MongoDB
المشروع يستخدم **MongoDB Atlas** (أو Local) لتخزين البيانات بشكل ديناميكي:
- **Users**: لتخزين بيانات الدخول (الاسم، الإيميل، كلمة المرور المشفرة بـ Bcrypt، والدور).
- **Workers**: لتخزين بيانات الصنايعية الإضافية (الحرفة، السعر، التقييم، سنوات الخبرة).
- **Requests**: لربط العملاء بالصنايعيين (يخزن ID العميل، ID الصنايعي، وصف المشكلة، الحالة، والوقت).

## 🚀 كيفية التشغيل (Setup)

1. **تشغيل الباك إند**:
   ```bash
   cd backend
   npm install
   npm run dev
   ```

2. **تشغيل التطبيق**:
   ```bash
   cd te
   flutter pub get
   flutter run -d chrome
   ```

- **Frontend**: Flutter (Dart)
- **Backend**: Node.js & Express
- **Database**: MongoDB & Mongoose
- **Authentication**: JWT (JSON Web Tokens)
- **Design**: Material Design 3 with custom gradients.
